# ТЗ: Полная переработка сайта **future-screen.ru** (новая версия на TS + Vite)

Документ предназначен для передачи в генеративный ИИ / разработчику, чтобы собрать **новый сайт** на современном стеке (TypeScript + Vite и т.д.) **с сохранением всей текущей информации**, но с исправлениями по структуре, офферам, UX и конверсии.

> Примечание по источникам: часть страниц сайта future-screen.ru ограниченно доступна для автоматического чтения, поэтому контент ниже собран из открыто доступных фрагментов/сниппетов и страниц, которые удалось получить. Всё, что не удалось подтвердить автоматически, помечено как **[НУЖНО ПОДТВЕРДИТЬ]** и должно быть сверено вручную (копипастой из админки/страниц).  
> Ключевые источники: главная, страницы услуг, “Комплексные решения”, “Светодиодное оборудование”, “Сцены и подиумы”, “Звуковое оборудование”.

## 1) Цели проекта

### 1.1 Бизнес-цели
- Сделать сайт, который воспринимается как **технический партнёр** (а не “каталог проката”).
- Упростить получение заявок: **КП/расчёт** → контакт → быстрый ответ.
- Подготовить сайт под performance-маркетинг: **Яндекс.Директ (поиск + ретаргет)**, VK-ретаргет, SEO.

### 1.2 Пользовательские цели
- За 10–20 секунд понимать: *что делаете, где работаете, что входит, как быстро, как связаться*.
- Быстро выбрать решение: LED / свет / звук / видео / сцена / “под ключ”.
- Понять доверие: опыт, формат мероприятий, наличие специалистов.

## 3) Что нужно исправить (обязательные правки)

1) **Переписать оффер и первый экран**: вместо общих слов — конкретно “что/где/как быстро/что входит/CTA”.
2) Вынести в верхний уровень навигации: **LED**, **Техсопровождение**, **Кейсы**, **Цены/Пакеты**, **Контакты**.
3) Убрать повторяющиеся блоки “Почему именно мы / Почему выбирают” со всех страниц: оставить 1 сильный блок доверия + факты.
4) Сделать **пакеты** (S/M/L или “Лайт/Медиум/Биг”) с составом и вилками (без точных цен можно, но нужна рамка).
5) Ввести нормальную структуру “кейсов” (с цифрами и контекстом). Сейчас **[НУЖНО ПОДТВЕРДИТЬ]** как оформлены “Наши работы”.
6) Везде заменить формулировку **«техобеспечение» → «техсопровождение»**.
7) Продумать бренд: минимум — подпись на русском под логотипом (без смены бренда) **[НУЖНО ПОДТВЕРДИТЬ юридически]**.

## 6) Контент-модель (без CMS на старте)

### 6.1 Кейсы (Markdown)
`content/cases/<slug>.md`  
Frontmatter:
- `title`, `city`, `date`, `format`

- `summary`
- `metrics` (опционально)
- `images[]`

### 6.2 Пакеты (JSON)
`data/packages.json`:
- `id`: light|medium|big
- `name`
- `for` (форматы)
- `includes`
- `options`
- `priceHint` (опционально)

### 6.3 Категории (JSON)
`data/categories.json`:
- `id`, `title`, `shortDescription`, `bullets[]`, `pagePath`

### 6.4 Контакты (JSON)
`data/contacts.json`:
- `phones[]`, `whatsappPhone`, `emails[]`, `address`, `workingHours`

## 7) Технологическое ТЗ (TS + Vite)

### 7.1 Стек
- Vite + React + TypeScript
- Router: react-router-dom
- Контент: markdown-to-jsx или MDX
- CSS: Tailwind (рекомендовано) или CSS Modules (выбрать один)
- Формы: react-hook-form + zod
- Хостинг: Cloudflare Pages / Netlify / Vercel

### 7.2 Перфоманс
- Lazy-load изображений
- WebP/AVIF
- Code splitting по роутам
- Lighthouse mobile >= 90 (цель)

### 7.3 SEO
- title/description на страницу
- OG-теги
- sitemap.xml + robots.txt
- structured data (Organization/LocalBusiness/BreadcrumbList)
- редиректы со старых URL

### 7.4 Аналитика
- Яндекс.Метрика + события:
  - `click_phone`, `click_whatsapp`, `submit_form`, `submit_quiz`, `view_case`
- готовность к коллтрекингу (скрипт + подмена номеров)

### 7.5 Заявки без CRM
- serverless endpoint (Cloudflare Worker / Netlify Function):
  - отправка в Telegram + email
  - антиспам (honeypot + rate limit)
  - сбор utm/referrer/pagePath
