# Contributing Guidelines

## Git Commit Messages

All commit messages must be written in **English**.

### Format

```
<type>: <short description>

[optional body]

[optional footer]
```

### Types

- `feat` — New feature
- `fix` — Bug fix
- `docs` — Documentation changes
- `style` — Code style changes (formatting, semicolons, etc.)
- `refactor` — Code refactoring
- `perf` — Performance improvements
- `test` — Adding or updating tests
- `chore` — Build process or auxiliary tool changes

### Examples

```
fix: resolve contact page routing issue

feat: add dark mode toggle

refactor: extract reusable Button component
```

### Rules

1. Use imperative mood ("add" not "added" or "adds")
2. Don't capitalize the first letter
3. No period at the end
4. Keep the first line under 72 characters
5. Write in English only

## Руководство по внесению изменений

### Сообщения коммитов

Все сообщения коммитов должны быть написаны на **английском языке**.

### Формат

```
<тип>: <краткое описание>

[опциональное тело]

[опциональный футер]
```

### Типы

- `feat` — Новая функциональность
- `fix` — Исправление ошибки
- `docs` — Изменения в документации
- `style` — Изменения стиля кода (форматирование, точки с запятой и т.д.)
- `refactor` — Рефакторинг кода
- `perf` — Улучшение производительности
- `test` — Добавление или обновление тестов
- `chore` — Изменения в сборке или вспомогательных инструментах

### Примеры

```
fix: resolve contact page routing issue

feat: add dark mode toggle

refactor: extract reusable Button component
```

### Правила

1. Используйте повелительное наклонение ("add", а не "added" или "adds")
2. Не пишите первую букву заглавной
3. Без точки в конце
4. Первая строка должна быть не длиннее 72 символов
5. Пишите только на английском языке
